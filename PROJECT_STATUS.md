## 4. 当前模型效果

基于 8,612 场比赛的回测结果：

| 预测任务 | 最佳模型 | 准确率 | AUC | 回测 ROI | 最大回撤 |
|----------|----------|--------|-----|----------|----------|
| **亚洲让球盘** | CatBoost | 76.11% | 0.856 | **+52.91%** | 4.03 单位 |
| **大小球** | XGBoost | 57.29% | 0.584 | **+11.34%** | 12.53 单位 |
| **胜平负** | CatBoost | 54.34% | 0.691 | **−2.63%** | 62.68 单位 |

**关键结论:**
- 亚洲盘模型表现优秀，高准确率 + 强 AUC + 低回撤，最具生产可用性。
- 大小球模型有正向优势，但 AUC 偏低，排序能力有限。
- 胜平负模型准确率优于随机基准 (33%)，但 ROI 为负，**平局预测是明显短板** (所有模型 F1 ≈ 0.07–0.19)。
- 最重要的特征因任务而异：亚盘方向 (`asian_direction`) 在亚洲盘任务中具有压倒性优势。

## 10. 重要配置说明

### 数据目录

所有数据统一存放在 **`D:/FootballData/`** 下，代码与数据分离：

```
D:/FootballData/
├── database/       ← SQLite 数据库
├── exports/        ← JSON/CSV 导出
├── raw_data/       ← 清洗后数据
├── snapshots/      ← 时间序列快照
└── models/         ← 训练好的 .pkl 模型
```

### 环境变量 (.env)

| 变量 | 说明 | 必填 |
|------|------|------|
| `FOOTBALL_DATA_ORG_API_KEY` | football-data.org API Key | 否 (免费套餐即可) |
| `DATA_ROOT=D:/FootballData` | 统一数据根目录 | 是 |
| `HEADLESS=true` | Playwright 无头模式 | 否 |
| `DATABASE_URL` | PostgreSQL 连接串 (可选) | 否 |

### Python 环境

- **版本**: Python 3.11.9 (Windows)
- **调用方式**: `py` 启动器 + `.venv` 虚拟环境
- **关键依赖**: Playwright, pandas, XGBoost, LightGBM, CatBoost, FastAPI

- `.env` / credentials 永不提交 (`.gitignore` 保护)
- 单文件 >1MB 不提交 (模型 .pkl、数据库 .db 已排除)
- 所有生成数据 (`datasets/`, `models/`, `data/`) 不进入版本控制
